# warp

## 关于

[Warp](https://www.warp.dev) 是一个源于终端的智能开发环境。您可以使用 Warp 内置的编码智能体，也可以自带 CLI 智能体（Claude Code、Codex、Gemini CLI 等）。

## 安装

您可以 [下载 Warp](https://www.warp.dev/download) 并查阅 [我们的文档](https://docs.warp.dev/) 获取特定平台的安装说明。

## Warp 贡献总览仪表板

访问 [build.warp.dev](https://build.warp.dev) 可以：
- 观看数千个 Oz 智能体对问题进行分类、编写规格说明、实现更改和审查 PR
- 查看热门贡献者和进行中的功能
- 通过 GitHub 登录跟踪您自己的 issue
- 在 Web 编译版 Warp 终端中点击进入活跃的智能体会话

## 许可证

Warp 的 UI 框架（`warpui_core` 和 `warpui` crate）采用 [MIT 许可证](LICENSE-MIT) 授权。

本仓库中的其余代码采用 [AGPL v3 许可证](LICENSE-AGPL) 授权。

## 开源与贡献

Warp 的客户端代码库是开源的，位于本仓库中。我们欢迎社区贡献，并设计了轻量级的工作流程来帮助新贡献者入门。完整的贡献流程请阅读我们的 [CONTRIBUTING.md](CONTRIBUTING.md) 指南。

### 从 Issue 到 PR

在提交 issue 之前，请先 [搜索已有 issue](https://github.com/warpdotdev/warp/issues?q=is%3Aissue+is%3Aopen+sort%3Areactions-%2B1-desc)，看看是否已存在相同的 Bug 或功能请求。如果没有找到，请使用我们的模板 [提交 issue](https://github.com/warpdotdev/warp/issues/new/choose)。安全漏洞请根据 [CONTRIBUTING.md](CONTRIBUTING.md#reporting-security-issues) 中的说明私下报告。

提交 issue 后，Warp 的维护者会进行审核，并可能添加就绪标签：[`ready-to-spec`](https://github.com/warpdotdev/warp/issues?q=is%3Aissue+is%3Aopen+label%3Aready-to-spec) 表示设计开放给贡献者来细化规格，[`ready-to-implement`](https://github.com/warpdotdev/warp/issues?q=is%3Aissue+is%3Aopen+label%3Aready-to-implement) 表示设计已定型，欢迎提交代码 PR。任何人都可以领取带有这些标签的 issue —— 如果您希望某个 issue 获得就绪标签，请在 issue 中提及 **@oss-maintainers**。

### 本地构建仓库

从源码构建和运行 Warp：

```bash
./script/bootstrap   # 平台相关设置
./script/run         # 构建并运行 Warp
./script/presubmit   # 格式化、clippy 检查及测试
```

完整的工程指南（包括代码风格、测试和平台特定说明）见 [WARP.md](WARP.md)。
