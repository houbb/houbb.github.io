# 博客书写规范和心德

写了大概3年多的博客，回首看收获了很多。

为了以后博客的更好管理，规定如下标准。

## 标题标准

时间：2022-05-29

发现 search 无法工作，debug 发现标题如果出现 `"`，会导致 json 的解析失败。

## url

统一采用:

```
yyyy-MM-dd-${category}-${order}-name-${order}.md
```

所有名称统一使用小写的形式。

所有同一个类型的博客，放在一起形成一个系列。

除非确定一定不会扩展的内容，建议还是学习所有的知识都是从 00-overview 开始。

## tag

常见的 tag 规定如下：

```
overview  概览
quick-start 快速开始
why-so-fast 为什么速度这么快
why-choose 为什么选择这个技术
project-architecture 项目架构
in-action 实战
summary 总结
sh 新知识(stay-hungry)
sf 旧知识(stay-foolish)
```

技术分类则根据实际需要


## 一遍文章的行文

作为技术文章，建议有以下几个部分：

1. 开篇：介绍技术的背景，实际的业务场景

2. 概念理论：

3. 实战测试：

4. 拓展发散：
